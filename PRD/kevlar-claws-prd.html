PRD: The "Zero-Trust" Refactor (Project: Kevlar-Claws)

1. Executive Summary

The goal is to transition cheetahclaws from a "Heuristic-Based" agent (guessing via string matching) to a "Verifiable" agent (confirming via Abstract Syntax Trees and State Checksums). We are moving the intelligence out of the LLM prompt and into the system's guardrails.

2. The Problem Statement

The current implementation suffers from Semantic Drift:

  • The String Matching Bug: Simple edits break file structure.
  • The Blind Trust Bug: The agent assumes the file on disk matches memory.
  • The "Shadow" Commit: AI changes are untraceable and often indistinguishable from human errors.

3. Functional Requirements

FR-01: Deterministic State Verification

FeatureRequirementImplementation
SnapshottingCompute SHA-256 hash before context injection.state_manager.py
Pre-Flight CheckRe-hash disk before execution.Middleware in tools/edit.py
Desync ActionAbort on hash mismatch.ERROR_STATE_DESYNC

FR-02: AST-Based Code Mutations

We are banning the use of re.sub or .replace(). The LLM will now interact with the code structure, not the text.

  • LibCST Integration: Mutations must be applied to specific Syntax Nodes.
  • Validation: If the CST does not compile, the tool reports failure immediately.

FR-03: Tiered Execution Permissions

  • Tier 1 (Read-Only): ls, grep, cat. (Auto-allowed)
  • Tier 2 (Surgical): Edit (AST only), Linter. (Auto-revert on fail)
  • Tier 3 (Destructive): Bash, Git Commit, RM. (Requires 95%+ Confidence Score or Human-in-the-Loop)

5. Success Metrics (QA KPI)

  1. Zero "Ghost Edits": 0% overwrites of changed files.
  2. Zero Syntax Regressions: 100% of AI-generated edits pass flake8/mypy.
  3. Traceability: 100% of AI-generated commits identifiable.

6. The "Angry QA" Anti-Requirements

  • No "Buddy" Pets: We are building a compiler-aware scalpel, not a cat.
  • No Stealth Mode: If the agent is used to "hide" AI code, self-destruct.
  • No Unbounded Context: Files >2000 lines require AST indexing.
